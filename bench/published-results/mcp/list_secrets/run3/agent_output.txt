The Supabase MCP server available here does not include a `list_secrets` (or equivalent) tool — the full tool list is:

`list_projects`, `get_project`, `list_organizations`, `get_organization`, `list_tables`, `list_migrations`, `list_branches`, `list_edge_functions`, `get_edge_function`, `execute_sql`, `generate_typescript_types`, `get_project_url`, `get_publishable_keys`, `get_advisors`, `get_logs`, `search_docs`, `list_extensions`

Project secrets/function environment variables are a **platform-level resource** (not stored in the database), so they're not queryable via `execute_sql` either. To list secret names you'd need one of:

- The **Supabase CLI**: `supabase secrets list --project-ref ebwcjhmrvwekjiybhytt`
- The **Supabase Management API**: `GET /v1/projects/{ref}/secrets` (requires a service role or management token)
- The **Supabase dashboard** → Settings → Edge Functions → Secrets

Since this environment has no shell access and the MCP server is read-only without a secrets tool, I'm unable to retrieve that list from here. You can run the CLI command above locally if you have the Supabase CLI installed and authenticated.
