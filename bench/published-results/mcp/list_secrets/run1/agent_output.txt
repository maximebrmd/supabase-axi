The registered Supabase MCP server does not include a `list_secrets` (or equivalent) tool. The full set of available tools is:

`list_projects`, `list_organizations`, `get_organization`, `get_project`, `get_project_url`, `get_publishable_keys`, `list_tables`, `list_migrations`, `list_branches`, `list_edge_functions`, `get_edge_function`, `list_extensions`, `execute_sql`, `generate_typescript_types`, `get_advisors`, `get_logs`, `search_docs`

None of these expose project secrets or function environment variable names. The Supabase Management API does have a `GET /v1/projects/{ref}/secrets` endpoint that would return secret names, but it is not surfaced through this MCP server instance.

To retrieve secret names you would need one of:
- **Supabase CLI**: `supabase secrets list --project-ref ebwcjhmrvwekjiybhytt`
- **Management API directly**: `GET https://api.supabase.com/v1/projects/ebwcjhmrvwekjiybhytt/secrets` with a service-role or management token
- An MCP server instance that includes the `list_secrets` tool
